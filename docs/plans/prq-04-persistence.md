# PRQ-04: Persistence

**Status:** queued

**Blocked by:** PRQ-03.

## Goal

Wire drizzle + dual-adapter SQLite (sql.js + jeep-sqlite on web; @capacitor-community/sqlite on native) + `@capacitor/preferences` so that floor seed, dirty chunks, placed structures, claimed water-coolers, kills, threat, inventory, weapons, journal, recipes-known all round-trip across reload. Migrations run on first boot per device. After this PRQ: `pnpm dev` survives a reload with full state recovery.

## Spec reference

§ 8 Persistence (locked), § 9 State split (locked), § 10 Threat system (locked), § 19 §19.3 Persistence bar.

## Success criteria

- `src/db/schema/index.ts` defines all 9 tables per spec §8.1.
- `src/db/client.ts` chooses the runtime adapter via `Capacitor.isNativePlatform()`.
- `src/db/migrations/0000_initial.sql` is generated by drizzle-kit, committed, replays cleanly on first boot.
- `src/db/repos/*.ts` provide typed CRUD for each table (no raw SQL outside repos).
- `src/db/preferences.ts` is a typed wrapper around `@capacitor/preferences` exposing all 8 keys from spec §8.5.
- Save loop debounced 1s, single transaction per flush; flush on `pagehide` / `blur` / `appStateChange`.
- Quit + relaunch round-trip preserves: seed, current_floor, threat, kills, claimed water-cooler, placed structures, dirty chunks, inventory, weapons, journal, recipes_known.
- `pnpm test:node` covers schema correctness + migration replay + every repo CRUD + transaction semantics. `pnpm test:browser` covers preferences round-trip + sql.js bootstrap-and-replay smoke.

## Task breakdown

### T1: Drizzle schema

**Files:** `src/db/schema/{world,chunks,structures,coolers,inventory,weapons,journal,recipes,kills}.ts`, `src/db/schema/index.ts` (re-exports).

Each table per spec §8.1. Use `integer` for floor/x/y/z, `blob` for `chunks.dirty_blob`, `text` for slugs, `integer({mode: 'timestamp_ms'})` for `*_at`.

**Acceptance:** `pnpm drizzle:gen` produces `src/db/migrations/0000_initial.sql`; `pnpm test:node` schema-shape test passes.

### T2: Migration runner + schema_version tracking

**Files:** `src/db/migrate.ts`.

On client boot: read `world_meta.schema_version`; replay migrations newer than that in order; commit new schema_version. Forward-only.

**Acceptance:** node test simulates 0→1 migration on a sql.js in-memory DB.

### T3: Web adapter (sql.js + jeep-sqlite)

**Files:** `src/db/client.web.ts`, `scripts/copy-wasm.mjs` (replace stub from PRQ-00 — copy `sql-wasm.wasm` into `public/wasm/`).

Bootstrap: lazy-import sql.js, initialize from `/wasm/sql-wasm.wasm`, register `jeep-sqlite` web component into the page once. Drizzle proxy adapter wraps the `sql.js` Database.

**Acceptance:** browser test mounts the adapter, runs migration, inserts/reads `world_meta`.

### T4: Native adapter (@capacitor-community/sqlite)

**Files:** `src/db/client.native.ts`.

Bootstrap via `CapacitorSQLite.createConnection`/`open`/`execute`. Same drizzle-proxy interface as web.

**Acceptance:** node test imports the module without side effects (full native validation deferred to PRQ-16 mobile shell).

### T5: Adapter dispatcher

**Files:** `src/db/client.ts`.

`Capacitor.isNativePlatform() ? import('./client.native') : import('./client.web')`. Returns a singleton `db` after migrations.

**Acceptance:** browser test: `await db.ready()` resolves; `db.select(world_meta)` returns initialized row.

### T6: Repos

**Files:** `src/db/repos/{world,chunks,structures,coolers,inventory,weapons,journal,recipes,kills}.ts`.

Typed CRUD per table. Examples: `chunks.upsert(floor, x, z, blob)`, `chunks.list(floor)`, `kills.increment(slug)`, `world.setThreat(value)`, `coolers.claim(floor, x, y, z)`.

**Acceptance:** node tests cover happy-path + concurrent-write per repo.

### T7: Preferences typed wrapper

**Files:** `src/db/preferences.ts`.

```ts
export const preferences = {
  get: async <K extends PrefKey>(k: K): Promise<PrefValue<K>> => ...,
  set: async <K extends PrefKey>(k: K, v: PrefValue<K>): Promise<void> => ...,
};
```
Keys: `volume_master`, `volume_sfx`, `volume_music`, `look_sensitivity`, `graphics_tier`, `world_seed`, `last_floor`, `controls_scheme`. Defaults baked in.

**Acceptance:** browser test: set/get round-trip on `@capacitor/preferences` (web shim).

### T8: Save loop wiring

**Files:** `src/db/save-loop.ts`, `src/ecs/save-bridge.ts`.

Subscribes to koota world changes (chunk dirty, inventory delta, cooler claim) → enqueues writes → debounced 1s batch flush via single transaction. Flush triggers: `pagehide`, `blur`, `Capacitor.App.addListener('appStateChange', ...)`. Idle flushes ≤ 1/s.

**Acceptance:** browser test simulates: dirty 5 chunks → wait 1.2s → assert single transaction wrote 5 chunks.

### T9: Round-trip integration test

**Files:** `e2e/persistence.spec.ts`.

Boot, set seed, dirty a chunk, claim a cooler, increment threat, reload page, assert all values recovered.

**Acceptance:** `pnpm test:e2e:ci` covers it; tagged `@golden`.

### T10: PR + merge

PR: `feat(persistence): drizzle + sql.js/capacitor-sqlite adapters + repos + save loop (PRQ-04)`. Squash-merge after `validate-deployed` green.
